# C - File I/O

This project focuses on the basics of File I/O in C, specifically how to manipulate files using Linux system calls.

## Learning Objectives
- Understanding file descriptors and their role in Linux.
- How to use system calls: `open`, `close`, `read`, and `write`.
- The difference between standard file descriptors (stdin, stdout, stderr).
- Managing file permissions and flags (O_RDONLY, O_WRONLY, O_RDWR, O_CREAT, etc.).
- The difference between standard library functions and system calls.

## Requirements
- Allowed editors: `vi`, `vim`, `emacs`.
- Compilation: `gcc -Wall -Werror -Wextra -pedantic -std=gnu89`.
- All code must follow the Betty style.
- No global variables allowed.
- Use only allowed syscalls (`open`, `close`, `read`, `write`) and `malloc`, `free`, `exit`.

## Project Tasks
- **0-read_textfile.c**: A function that reads a text file and prints it to the POSIX standard output.
